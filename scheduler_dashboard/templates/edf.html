Earliest Deadline First (EDF)

Optimal Dynamic Real-Time Scheduling

View Source Code

Introduction

Earliest Deadline First (EDF) is a dynamic priority real-time scheduling algorithm. At any point in time, the process with the closest absolute deadline gets the highest priority and runs first.

Optimality:

EDF is provably optimal for single-processor real-time systems. If ANY algorithm can schedule a set of tasks without missing deadlines, EDF can too!

Mathematical Formulas

Deadline Calculation

Absolute Deadline = Arrival Time + Relative Deadline
Priority is inversely proportional to absolute deadline. Closest deadline = highest priority.

Schedulability Test

U = Σ(Ci / Ti) ≤ 1.0 (100%)
Total CPU utilization must not exceed 100%. EDF can achieve full utilization!

Utilization Example

Given periodic tasks:

  • Task A: C=1ms, T=4ms → U_A = 1/4 = 0.25
  • Task B: C=2ms, T=6ms → U_B = 2/6 = 0.33
  • Task C: C=3ms, T=12ms → U_C = 3/12 = 0.25

Total U = 0.25 + 0.33 + 0.25 = 0.83 ≤ 1.0 ✓ Schedulable!

Advantages & Disadvantages

Advantages

  • Optimal for single-processor
  • Can achieve 100% utilization
  • Works with periodic/aperiodic tasks
  • No starvation
  • Minimizes deadline misses

❌ Disadvantages

  • Higher runtime overhead
  • Domino effect when overloaded
  • Harder to analyze than RMS
  • Requires accurate deadlines
