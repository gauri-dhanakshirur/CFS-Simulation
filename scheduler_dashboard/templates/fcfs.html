First Come First Serve (FCFS)

The Simplest CPU Scheduling Algorithm

View Source Code

Introduction to FCFS

First Come First Serve (FCFS) is the simplest CPU scheduling algorithm. It works exactly like a queue at a bank or grocery store: the process that arrives first gets served first, regardless of how long it takes to complete.

FCFS is a non-preemptive algorithm, meaning once a process starts executing, it runs to completion without interruption. No process can "cut in line" — even if a very short task arrives while a long task is running.

Key Property:

FCFS is non-preemptive and uses a FIFO (First-In-First-Out) queue. The order of arrival completely determines the execution order.

Mathematical Formulas

Waiting Time

Waiting Time (WT) = Start Time - Arrival Time
The time a process spends waiting in the ready queue before it starts executing

Turnaround Time

Turnaround Time (TAT) = Completion Time - Arrival Time
Total time from arrival to completion = Waiting Time + Burst Time

Response Time

Response Time (RT) = First Run Time - Arrival Time
In FCFS, Response Time = Waiting Time (process runs to completion once started)

Average Metrics

Average WT = Σ(WTi) / n
Sum of all waiting times divided by number of processes

Step-by-Step Algorithm

1 Initialize Queue

Create an empty FIFO queue to hold arriving processes.

2 Enqueue Arrivals

As processes arrive, add them to the back of the queue in arrival order.

3 Dequeue & Execute

Take the process at the front of the queue and execute it completely.

4 Record Metrics

When process completes, calculate its WT and TAT.

5 Repeat Until Empty

Continue until all processes are complete and queue is empty.

Detailed Example

Problem Setup

Given processes with the following arrival and burst times:

Process Arrival Time Burst Time
P1 0 24
P2 1 3
P3 2 3

Gantt Chart

P1 (0-24)
P2 (24-27)
P3 (27-30)
0 24 27 30

Step-by-Step Calculations

Process P1

  • Arrival Time (AT) = 0
  • Start Time = 0 (CPU is idle, P1 starts immediately)
  • Completion Time (CT) = 0 + 24 = 24
  • Turnaround Time = CT - AT = 24 - 0 = 24
  • Waiting Time = TAT - BT = 24 - 24 = 0

Process P2

  • Arrival Time (AT) = 1
  • Start Time = 24 (must wait for P1 to complete)
  • Completion Time (CT) = 24 + 3 = 27
  • Turnaround Time = CT - AT = 27 - 1 = 26
  • Waiting Time = TAT - BT = 26 - 3 = 23

Process P3

  • Arrival Time (AT) = 2
  • Start Time = 27 (must wait for P1 and P2)
  • Completion Time (CT) = 27 + 3 = 30
  • Turnaround Time = CT - AT = 30 - 2 = 28
  • Waiting Time = TAT - BT = 28 - 3 = 25

Final Results Table

Process AT BT CT TAT WT
P1 0 24 24 24 0
P2 1 3 27 26 23
P3 2 3 30 28 25
Averages 26 16

Average Waiting Time = (0 + 23 + 25) / 3 = 48/3 = 16 ms

Average Turnaround Time = (24 + 26 + 28) / 3 = 78/3 = 26 ms

The Convoy Effect

Major Problem with FCFS:

When a long process arrives first, all shorter processes must wait behind it. This is called the "Convoy Effect" — like being stuck behind a slow truck on a single-lane road!

In our example above, P2 and P3 (each needing only 3ms) had to wait 23 and 25 ms respectively because P1 (needing 24ms) arrived first. If we had used SJF instead:

What if we used SJF instead?

  • P2 executes: 1-4 (WT=0)
  • P3 executes: 4-7 (WT=2)
  • P1 executes: 7-31 (WT=7)

Average WT with SJF = (0 + 2 + 7)/3 = 3 ms (vs 16 ms with FCFS!)

Advantages & Disadvantages

Advantages

  • Extremely simple to implement
  • No complex calculations needed
  • No starvation — all processes eventually run
  • Fair in terms of arrival order
  • Zero scheduling overhead
  • Predictable, deterministic behavior

Disadvantages

  • Convoy Effect — poor average wait time
  • Not suitable for interactive systems
  • Cannot prioritize important processes
  • Long response time for short processes
  • Poor CPU utilization with I/O-bound processes
  • Non-preemptive — no responsiveness

When to Use FCFS

Use FCFS When... Don't Use FCFS When...
Simple batch processing systems Interactive/time-sharing systems
All processes have similar burst times Workloads have varying burst times
Order of execution doesn't matter Response time is critical
Print queues, background jobs Real-time or GUI applications
