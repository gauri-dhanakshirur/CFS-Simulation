CPU Scheduling Algorithm Simulator

An interactive visualization platform for understanding and comparing CPU scheduling algorithms used in operating systems.

About This Project

This simulator provides a hands-on way to explore how different CPU scheduling algorithms manage process execution. The project was built to bridge the gap between theoretical operating systems concepts and practical understanding.

Powered by a high-performance C backend, the simulator executes real scheduling algorithms and visualizes their behavior through interactive Gantt charts, performance metrics, and fairness analysis.

Purpose & Use Cases

Educational Tool

Perfect for students studying Operating Systems courses. Visualize how algorithms like FCFS, SJF, Round Robin, and CFS actually schedule processes.

  • See Gantt charts animate process execution
  • Understand waiting time and turnaround time calculations
  • Compare algorithm trade-offs visually

Developer Benchmarking

Useful for OS and system developers who need to compare different scheduling strategies and their metrics.

  • Analyze performance metrics like throughput and CPU utilization
  • Evaluate Jain's Fairness Index for scheduler fairness
  • Compare multiple algorithms side-by-side

Algorithms Implemented

The simulator implements 9 different CPU scheduling algorithms, from basic to advanced:

FCFS First Come First Serve SJF Shortest Job First RR Round Robin Priority Priority-Based MLFQ Multi-Level Feedback Queue CFS Linux's Completely Fair Scheduler EDF Earliest Deadline First RMS Rate Monotonic Scheduling PropShare Lottery Scheduling

How It Works

1

Select an Algorithm

Choose a scheduling algorithm from the sidebar. Each algorithm has specific parameters and characteristics.

2

Configure Processes

Add processes with arrival time, burst time, and algorithm-specific parameters (priority, deadline, tickets, etc.).

3

Run Simulation

Click "Run Simulation" to execute the scheduler. The C backend processes your input and generates execution data.

4

Analyze Results

View the Gantt chart, performance metrics (WT, TAT, RT), CPU utilization, throughput, and fairness index.

Metrics Explained

Waiting Time (WT)

Time a process spends waiting in the ready queue before execution.

Turnaround Time (TAT)

Total time from process arrival to completion (WT + Burst Time).

Response Time (RT)

Time from process arrival to first CPU allocation.

CPU Utilization

Percentage of time the CPU is actively executing processes.

Throughput

Number of processes completed per unit time.

Jain's Fairness Index

A value between 0-1 measuring how fairly CPU time is distributed (1 = perfect fairness).
