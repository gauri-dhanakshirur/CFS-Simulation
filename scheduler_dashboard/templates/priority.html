Preemptive Priority Scheduling

Scheduling Based on Process Importance

View Source Code

Introduction

Priority Scheduling assigns a priority number to each process. The CPU is allocated to the process with the highest priority. In the preemptive variant, if a new process arrives with higher priority, the current process is preempted.

Convention:

Lower number = higher priority (priority 0 is highest). Some systems use opposite convention.

Mathematical Formulas

Next Process = arg min(Priority Number)
Select process with lowest priority number (highest priority)
Preempt if: Priority(new) < Priority(current)
Preempt when newly arrived process has higher priority

Aging Formula (Anti-Starvation)

Effective Priority = Base Priority - ⌊Wait Time / Age Factor⌋
Priority improves over time to prevent starvation

Priority Levels

Priority Category Examples
0 (Highest) System Critical Interrupt handlers, kernel
1-3 Real-Time Audio/video, drivers
4-6 Interactive User apps, GUI, browsers
7-8 Batch Compilation, backups
9 (Lowest) Idle Screen savers, maintenance

Starvation & Priority Inversion

Starvation:

Low priority processes may never execute if high priority processes keep arriving.

Priority Inversion:

High-priority process waits for resource held by low-priority process, while medium-priority runs.

Solution: Aging

Gradually increase priority of waiting processes:
new_priority = base_priority - (wait_time / aging_factor)

Advantages & Disadvantages

Advantages

  • Important processes get more CPU
  • Good for real-time systems
  • Flexible priority assignment
  • User control over importance

Disadvantages

  • Starvation without aging
  • Priority inversion problem
  • Complex priority assignment
  • Context switch overhead
