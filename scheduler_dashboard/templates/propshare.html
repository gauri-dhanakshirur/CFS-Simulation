Proportional Share / Lottery Scheduling

Fair Share CPU Allocation with Randomization

View Source Code

Introduction

Proportional Share Scheduling (also called Lottery Scheduling) allocates CPU time in proportion to the number of "tickets" each process holds. More tickets = higher probability of being selected.

Instead of strict priorities, this scheduler uses randomization to achieve fairness over time. A process with 75% of tickets will, on average, get 75% of CPU time.

Mathematical Formulas

CPU Share

Sharei = Ticketsi / Σ Tickets
Each process gets CPU proportional to its ticket fraction

Win Probability

P(process i wins) = Ticketsi / Total Tickets
Probability of winning each lottery = share of total tickets

Expected Execution Time

Expected Time = Runtime × (Total / My Tickets)
Process with fewer tickets takes longer to complete (on average)

Advanced Mechanisms

Ticket Transfer

Processes can temporarily give tickets to other processes. A client can transfer tickets to a server while waiting for a response.

Ticket Inflation

In a trusted environment, processes can temporarily increase their tickets to get more CPU time when needed.

Ticket Currency

Groups of processes can have their own local ticket currency, converted to global tickets. A user with 100 global tickets can distribute them as "500 local tickets" among their jobs.

Advantages & Disadvantages

Advantages

  • Simple to understand and implement
  • Flexible resource allocation
  • No starvation (all have chance)
  • Probabilistic fairness
  • Supports ticket transfer/inflation

Disadvantages

  • Non-deterministic (randomness)
  • Requires many draws for fairness
  • Not suitable for real-time
  • Can have unfair short-term variance
