Rate Monotonic Scheduling (RMS)

Optimal Fixed-Priority Real-Time Scheduling

View Source Code

Introduction

Rate Monotonic Scheduling (RMS) is a static-priority real-time scheduling algorithm. The priority of each task is assigned based on its period: shorter period = higher priority. Priorities are fixed at design time and never change.

Optimality:

RMS is optimal among all fixed-priority algorithms. If any fixed-priority algorithm can schedule a task set, RMS can too!

Mathematical Formulas

Priority Assignment

Priority ∝ 1/Period
Task with shorter period gets higher priority (fixed at creation)

Liu & Layland Schedulability Bound

U = Σ(Ci/Ti) ≤ n(21/n - 1)
Where n = number of tasks, C = execution time, T = period

Utilization Bound Table

n (tasks) Bound n(21/n - 1) Percentage
1 1.000 100%

n → ∞ ln(2) = 0.693 69.3%
Key Insight:

RMS can only guarantee schedulability up to ~69% CPU utilization for large n. EDF can achieve 100%!

Advantages & Disadvantages

Advantages

  • Simple fixed-priority assignment
  • Easy to implement
  • Low runtime overhead
  • Easy to analyze offline
  • Optimal for fixed-priority

Disadvantages

  • Only ~69% max utilization
  • Requires known periods
  • Not flexible for aperiodic tasks
  • Priority inversion possible
